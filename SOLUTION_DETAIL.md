# Direction 1.5: 구글 스프레드시트 + 파이썬 자동화 솔루션 상세 가이드

## 1. 솔루션 개요

### 왜 "Direction 1.5"인가?

| 방향 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **방향 1** | 기존 시스템 유지 + 카톡 자동화 | 변화 최소 | 카톡 API로 파일 직접 전송 불가 |
| **방향 2** | 자체 통합 플랫폼 구축 | 완전한 커스터마이징 | 개발 비용 막대, 유지보수 부담 |
| **방향 1.5 (채택)** | 구글 시트 + 알림톡 조합 | 저비용, 빠른 구축, 안정적 | 구글 시트 의존성 |

**Direction 1.5**는 "방향 2"의 핵심 장점(웹 기반 송장 입력)을 가져오면서, 복잡한 개발 없이 **기존 무료 도구(Google Sheets)**를 활용하는 "가성비 최적화" 전략입니다.

### 핵심 아이디어
- **엑셀 파일을 주고받지 않는다** → 구글 시트 링크를 공유한다.
- **카톡으로 송장번호를 받지 않는다** → 업체가 직접 시트에 입력한다.
- **수동 복사-붙여넣기를 하지 않는다** → 파이썬이 자동으로 처리한다.

## 3. 상세 프로세스

### 3.1 환경 세팅 (최초 1회)

#### A. 구글 스프레드시트 구조 설계

**마스터 시트 (관리자 전용)**
- 용도: 전체 주문 현황 통합 조회, 송장 입력 현황 모니터링
- 권한: 관리자만 접근 가능

**업체별 시트 (위탁업체 공유용)**
- 파일 수: 위탁 업체 수만큼 (예: 30개)
- 파일명 예시: `[갓샵] A업체_발주서`, `[갓샵] B업체_발주서`
- 권한 설정:
  - 해당 업체 담당자 이메일에만 '편집자' 권한 부여
  - 또는 '링크가 있는 모든 사용자 편집 가능' (보안 낮음, 편의성 높음)

**시트 칼럼 구조 예시:**

| 주문일자 | 주문번호 | 수취인명 | 연락처 | 주소 | 상품명 | 옵션 | 수량 | 택배사 | 송장번호 |
|---------|---------|---------|--------|------|--------|------|------|--------|---------|
| 2024-01-28 | ORD001 | 홍길동 | 010-xxxx-xxxx | 서울시... | 텀블러 | 블랙 | 2 | (드롭다운) | (입력칸) |

- **송장번호 칸**: 노란색 배경으로 강조, 업체가 여기에만 입력
- **나머지 칼럼**: 시트 보호 기능으로 잠금 (수정 불가)

#### B. 구글 API 설정

1. [Google Cloud Console](https://console.cloud.google.com) 접속
2. 새 프로젝트 생성
3. "Google Sheets API" 및 "Google Drive API" 활성화
4. 서비스 계정(Service Account) 생성
5. JSON 키 파일 다운로드 → 파이썬 코드에서 사용

#### C. 알림톡 서비스 가입

**추천 서비스 비교:**

| 서비스 | 건당 비용 | 특징 |
|--------|----------|------|
| 알리고 (Aligo) | 약 9~15원 | 국내 점유율 높음, 문서 풍부 |
| 솔라피 (Solapi) | 약 10~15원 | 개발자 친화적 API |
| NHN Cloud | 약 10~12원 | 대기업 인프라 |

**필요한 것:**
- 사업자등록증 (알림톡 발신 프로필 등록용)
- 카카오 비즈니스 채널 개설
- API Key 발급

### 3.2 발주 자동화 (매일 아침 루틴)

**실행 시점:** 이지어드민에서 당일 주문 엑셀을 다운받은 직후

#### Step 1: 엑셀 파일 읽기

```
[입력] 이지어드민_발주서_20240128.xlsx
         │
         ▼
    ┌─────────────┐
    │ pandas로    │
    │ 데이터 로드  │
    └─────────────┘
         │
         ▼
[메모리] DataFrame (전체 주문 데이터)
```

**처리 내용:**
- 엑셀 파일을 pandas DataFrame으로 로드
- 날짜, 주문번호 등 데이터 정제
- 필요 시 칼럼명 표준화

#### Step 2: 업체별 데이터 분류

```
[전체 주문 100건]
         │
         │ '공급처' 또는 '브랜드' 컬럼 기준 필터링
         ▼
    ┌─────────────┐
    │ A업체: 25건  │
    │ B업체: 30건  │
    │ C업체: 15건  │
    │ D업체: 20건  │
    │ E업체: 10건  │
    └─────────────┘
```

**처리 내용:**
- `groupby()` 함수로 업체별 데이터 분리
- 각 업체별 DataFrame 생성

#### Step 3: 구글 시트 업데이트

```
A업체 DataFrame
         │
         ▼
    ┌─────────────────────────┐
    │ gspread 라이브러리      │
    │ A업체 구글 시트 접속     │
    │ 기존 데이터 삭제         │
    │ 새 데이터 붙여넣기       │
    │ 서식 적용 (노란색 칸 등) │
    └─────────────────────────┘
         │
         ▼
[결과] A업체 구글 시트 업데이트 완료
```

**처리 내용:**
- 각 업체의 고정 URL 구글 시트에 접속
- 이전 데이터 클리어 후 새 주문 데이터 삽입
- '송장번호' 칸은 비워두고 배경색 노란색 처리

#### Step 4: 알림톡 발송

```
업체 정보 (이름, 전화번호, 시트URL)
         │
         ▼
    ┌─────────────────────────┐
    │ 알림톡 API 호출          │
    │                         │
    │ [갓샵] 발주 알림         │
    │ 1월 28일 신규 주문 25건  │
    │ 이 등록되었습니다.       │
    │                         │
    │ ▶ 발주 확인하기 (버튼)   │
    │   → 구글 시트 링크       │
    └─────────────────────────┘
         │
         ▼
[결과] 업체 사장님 카톡으로 알림 도착
```

**알림톡 메시지 템플릿 예시:**
```
[갓샵] 발주 알림

안녕하세요, #{업체명} 사장님.
#{날짜} 신규 주문 #{건수}건이 등록되었습니다.

아래 링크에서 주문 내역 확인 후,
'송장번호' 칸에 입력 부탁드립니다.

※ 발송 완료 후 별도 연락은 필요 없습니다.
```

### 3.3 송장 수집 자동화 (매일 저녁 루틴)

**실행 시점:** 택배 마감 이후 (보통 오후 4~6시)

#### Step 1: 전체 시트 스캔

```
    ┌─────────────────────────┐
    │ 30개 업체 시트 순회      │
    │                         │
    │ A업체 시트 → 데이터 읽기 │
    │ B업체 시트 → 데이터 읽기 │
    │ C업체 시트 → 데이터 읽기 │
    │ ...                     │
    └─────────────────────────┘
```

#### Step 2: 송장번호 추출 및 검증

```
각 시트의 데이터
         │
         ▼
    ┌─────────────────────────┐
    │ 송장번호 칸 확인         │
    │                         │
    │ ✓ 입력됨 & 유효 → 수집   │
    │ ✗ 미입력 → 스킵          │
    │ ⚠ 형식 오류 → 로그 기록  │
    └─────────────────────────┘
```

**검증 로직:**
- 숫자만 있는지 확인
- 자릿수 체크 (택배사별 송장번호 길이 검증)
- 중복 송장번호 체크

#### Step 3: 이지어드민용 엑셀 생성

```
수집된 송장 데이터
         │
         ▼
    ┌─────────────────────────┐
    │ 이지어드민 양식에 맞춰   │
    │ 엑셀 파일 생성           │
    │                         │
    │ 컬럼: 주문번호, 택배사,  │
    │       송장번호           │
    └─────────────────────────┘
         │
         ▼
[출력] 송장일괄등록_20240128.xlsx
```

#### Step 4: 최종 업로드

**관리자 액션:**
- 생성된 `송장일괄등록_20240128.xlsx` 파일을 이지어드민에 업로드
- 일괄 발송 처리 완료

## 5. 예상 문제 및 해결책

### Q1: 업체 사장님이 구글 시트를 못 쓰면?

**해결책:**
- 엑셀과 사용법이 거의 동일함을 안내
- "링크 클릭 → 노란색 칸에 숫자만 입력" 가이드 이미지 제작
- 모바일: Google Sheets 앱 설치 권장 (폰에서도 쉽게 입력 가능)

### Q2: 업체가 실수로 주문 정보를 지우면?

**해결책:**
- **시트 보호 기능** 활용
- '송장번호', '택배사' 칼럼만 편집 가능하게 설정
- 나머지 칼럼(주문자명, 주소 등)은 잠금 처리

### Q3: 주문 취소가 발생하면?

**해결책 (초기 버전):**
- 100% 자동화보다는 수동 개입 허용
- 취소 건 발생 시 관리자가 해당 업체 시트에서 직접 행 삭제 또는 '취소' 표시
- 추후 고도화: 이지어드민 취소 데이터를 읽어서 자동 반영

### Q4: 업체가 송장번호를 이상하게 입력하면?

**해결책:**
- 송장번호 칼럼에 **데이터 유효성 검사** 적용
- 숫자만 입력 가능, 특정 자릿수만 허용
- 파이썬에서 2차 검증 후 오류 로그 생성 → 관리자에게 알림

### Q5: 인터넷이 끊기거나 API 오류가 나면?

**해결책:**
- 모든 API 호출에 **재시도 로직(Retry)** 적용
- 실패 건은 로그 파일에 기록
- 실행 완료 시 성공/실패 요약 리포트 출력

## 6. 비용 분석

### 월간 예상 비용 (30개 업체, 일 100건 주문 기준)

| 항목 | 단가 | 월간 비용 |
|------|------|----------|
| 구글 스프레드시트 | 무료 | 0원 |
| 구글 API 호출 | 무료 (일일 한도 내) | 0원 |
| 알림톡 발송 (발주 알림) | 15원/건 × 30업체 × 30일 | 약 13,500원 |
| 알림톡 발송 (미입력 리마인드, 선택) | 15원/건 × 10건 × 30일 | 약 4,500원 |
| **합계** | | **약 18,000원/월** |

**절감 효과:**
- 기존: 하루 1~2시간 × 30일 = 월 30~60시간 수작업
- 도입 후: 하루 10분 × 30일 = 월 5시간
- **월 25~55시간 절감** (시급 1만원 환산 시 25~55만원 가치)
