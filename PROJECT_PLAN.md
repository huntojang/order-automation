# 발주 송장 자동화 프로젝트 (Project Plan)

## 1. 프로젝트 개요
*   **목표:** 위탁 판매 업무의 비효율(카톡 수동 발주, 송장 수기 입력)을 자동화하여 운영 리소스 절감 및 휴먼 에러 방지.
*   **핵심 전략 (Direction 1.5):** 무거운 자체 플랫폼 구축 대신 **[구글 스프레드시트 + 파이썬 + 카카오 알림톡]** 조합을 활용한 실용적 자동화.

## 2. 현황 분석 (AS-IS)
*   **구조:** 자사몰(갓샵)이 주문을 받고(이지어드민), 30여 개 위탁 업체에 발주를 넣는 구조.
*   **문제점:**
    *   **발주:** 매일 엑셀을 쪼개서 수십 개 업체 카톡방에 일일이 파일 전송.
    *   **송장:** 업체가 카톡으로 보낸 송장번호를 보고 관리자가 수기로 이지어드민에 입력.
    *   **비효율:** 단순 반복 작업에 과도한 시간 소요, 오타 발생 위험.

## 3. 해결 방안 (TO-BE)
*   **아키텍처:** Python Script ↔ Google Sheets ↔ Kakao AlimTalk ↔ 위탁업체
*   **채택 방식:**
    1.  **발주 데이터베이스화:** 이지어드민 엑셀을 파이썬으로 파싱하여 '구글 스프레드시트'에 업체별로 자동 업로드.
    2.  **알림톡 전송:** "발주가 등록되었습니다" 알림톡 발송 (엑셀 파일 첨부 X, 시트 링크 O).
    3.  **웹 기반 입력:** 위탁 업체 사장님은 구글 시트 링크에 접속하여 '송장번호' 칸에 숫자만 입력.
    4.  **자동 취합:** 파이썬이 시트를 읽어 송장번호를 수집하고, 이지어드민 업로드용 엑셀 파일 자동 생성.

## 4. 상세 구현 프로세스

### 1단계: 환경 구축 (Setup)
*   **마스터 시트:** 관리자용 통합 데이터 시트.
*   **업체별 시트:** 30개 업체별 전용 시트 생성 및 공유 권한 설정 (업체 사장님 편집 가능).

### 2단계: 발주 자동화 (Daily Morning)
1.  이지어드민에서 엑셀 다운로드.
2.  파이썬 스크립트 실행 → 업체별 데이터 분류.
3.  각 업체별 구글 시트에 주문 데이터(수취인, 상품명, 수량 등) 업데이트.
4.  업체 사장님에게 알림톡(링크 포함) 자동 발송.

### 3단계: 송장 수집 자동화 (Daily Evening)
1.  파이썬 스크립트 실행 → 30개 구글 시트 순회.
2.  입력된 송장번호 추출 및 유효성 검증.
3.  이지어드민 양식에 맞는 '송장 일괄 등록용 엑셀' 파일 생성.
4.  관리자가 이지어드민에 최종 업로드.

## 5. 역할 분담
*   **AI (Cursor):**
    *   전체 파이썬 자동화 코드 작성 (Pandas, Gspread, API 연동).
    *   구글 클라우드 및 API 설정 상세 가이드 제공.
    *   오류 처리 및 안정성 로직 구현.
*   **사용자 (CEO):**
    *   알림톡 중개 업체(알리고, 솔라피 등) 가입 및 API Key 발급.
    *   구글 API 서비스 계정 설정 (가이드에 따라 진행).
    *   테스트용 샘플 데이터(이지어드민 엑셀, 업체 리스트) 제공.

## 6. 다음 진행 단계 (Next Steps)
1.  **샘플 데이터 준비:** 이지어드민 발주서 양식 & 위탁 업체 리스트 확인.
2.  **기술 검증:** 파이썬으로 엑셀 읽기 및 구글 시트 연동 테스트.
3.  **파일럿 테스트:** 전체 적용 전, 소통이 원활한 1~2개 업체와 선행 테스트 진행.
